🌊 Topic 1: Network Flow — What is it?

Imagine a water pipe network. Water flows from a source (like a reservoir) to a sink (like your house). Each pipe has a capacity — a maximum amount of water it can carry. The question is: how much water can reach your house?

REAL LIFE EXAMPLE Internet Traffic: Data packets flow from server (source) → your computer (sink). Each wire has a bandwidth limit (capacity). Network flow finds the maximum data transfer rate.

Key Terms (Plain English):

  • Source (s) — Where things start flowing FROM
  • Sink (t) — Where things flow TO (destination)
  • Capacity c(u,v) — Max flow allowed on edge u→v
  • Flow f(u,v) — Actual flow on edge (must be ≤ capacity)
  • Residual Graph — Shows remaining capacity after some flow
  • Augmenting Path — A path from s to t where we can push more flow
  • Max Flow — The maximum total flow from s to t

3 Rules of Flow:

RULE 1 — CAPACITY Flow on edge ≤ Capacity of edge
0 ≤ f(u,v) ≤ c(u,v)
RULE 2 — CONSERVATION At every node (except s and t): flow in = flow out (no "piling up")
RULE 3 — SKEW SYMMETRY f(u,v) = −f(v,u) (flow from A to B is opposite of B to A)
🤔 Topic 2: P vs NP — The Biggest Mystery in CS

Imagine two types of puzzles:

CLASS P — "EASY TO SOLVE" P = Polynomial Time
Problems we can solve quickly (in polynomial time = n², n³, etc.)

Real Example: Sorting a list of numbers. For n numbers, it takes roughly n×log(n) steps. Fast! ✅
CLASS NP — "EASY TO VERIFY" NP = Non-deterministic Polynomial
Problems where if someone gives you the answer, you can check it quickly, but finding it might take forever.

Real Example: Sudoku. Hard to solve, but easy to check if solution is correct. ✅
THE BIG QUESTION (Worth $1,000,000!) Does P = NP? Nobody knows. If yes, every problem easy to check is also easy to solve. Most believe P ≠ NP.
NP-COMPLETE — THE HARDEST IN NP A problem is NP-Complete if:
1. It is in NP (answers can be verified quickly)
2. Every other NP problem can be reduced to it (it's "as hard as any NP problem")
NP-HARD — EVEN HARDER At least as hard as NP-Complete, but may not even be in NP (you can't even verify quickly).
🎯 Topic 3: Approximation Algorithms — Close Enough!

Since NP-Complete problems are too hard to solve exactly, we accept a "good enough" answer. That's an Approximation Algorithm.

ANALOGY You need to travel to 10 cities cheaply (Travelling Salesman Problem — NP-Hard). Finding the PERFECT route is too hard. An approximation says: "Here's a route that costs at most TWICE the best possible." That's a 2-approximation.

Approximation Ratio (ρ): If optimal = OPT and our answer = C, then ρ = C/OPT. Lower is better. ρ=1 means perfect. ρ=2 means at most 2× the optimal cost.

🔍 Topic 4: Local Search — Improve Step by Step

Start with any solution. Look at "nearby" solutions. If a neighbor is better, move there. Repeat until no improvement is possible.

ANALOGY You're hiking and want the highest peak. Local search = always walk uphill. Problem: you might get stuck on a small hill (local optimum), not the highest mountain (global optimum).

Key idea: Local search finds local optima, not guaranteed global optima.

🎲 Topic 5: Randomization — Use Luck to Your Advantage

Sometimes adding randomness makes algorithms faster or simpler on average, even if the worst case is bad.

EXAMPLE — RANDOMIZED QUICKSORT Pick a random pivot instead of always first element. Avoids worst-case O(n²). Expected time: O(n log n).
EXAMPLE — RANDOMIZED SELECT Find the k-th largest element using random splitters. Works in O(n) expected time.

🌊 Ford-Fulkerson Algorithm (Max Flow)
  • 1
    Initialize: Set f(u,v) = 0 for all edges. The residual graph initially = original graph.
  • 2
    Find Augmenting Path: Use BFS/DFS to find any path from source (s) to sink (t) in the residual graph.
  • 3
    Find Bottleneck: The bottleneck = minimum residual capacity along the path. This is how much extra flow we can push.
  • 4
    Update: Add flow along the path. Update residual capacities: forward edges decrease, backward edges increase.
  • 5
    Repeat until no augmenting path exists.
Max Flow = Sum of flows out of source s
⚠️ COMMON MISTAKE Students forget to add backward edges in the residual graph. A backward edge from v→u with capacity f(u,v) allows us to "cancel" previous flow decisions!
MAX FLOW = MIN CUT THEOREM (KEY THEOREM!) The maximum flow from s to t equals the minimum cut capacity between s and t.
Cut: A partition of nodes into sets S (containing s) and T (containing t). Cut capacity = sum of capacities of edges crossing from S to T.
AlgorithmTime ComplexityMethod
Ford-FulkersonO(E × max_flow)DFS for augmenting paths
Edmonds-KarpO(V × E²)BFS for augmenting paths

Step 2: Reduce a known NPC problem to your problem in polynomial time
⚡ SAT, 3-SAT, CNF — Circuit Satisfiability

Key Definitions:

  • Literal: A variable (x) or its negation (¬x)
  • Clause: OR of literals: (x₁ ∨ ¬x₂ ∨ x₃)
  • CNF: AND of clauses: (clause1) ∧ (clause2) ∧ ...
  • SAT: Is there an assignment making CNF = TRUE?
  • 3-SAT: SAT where each clause has exactly 3 literals
  • Certificate: An assignment that satisfies the formula

Important Reductions:

  • SAT → 3-SAT → Independent Set
  • 3-SAT → Vertex Cover
  • 3-SAT → Clique
  • Independent Set → Vertex Cover (complement)
  • Vertex Cover → Set Cover
KEY FACT Vertex Cover of size k ⟺ Independent Set of size n-k
// 3-SAT → Independent Set Reduction Given: 3-SAT formula with k clauses Build graph: - For each clause (a ∨ b ∨ c), create 3 nodes: a, b, c - Connect all 3 nodes within each clause (triangle) - Connect node x in clause i to ¬x in clause j (conflict edges) Claim: 3-SAT is satisfiable ⟺ Graph has Independent Set of size k
🎯 Approximation Algorithms — Detailed
VERTEX COVER APPROX (2-approx) Algorithm:
While there are uncovered edges:
  Pick any uncovered edge (u,v)
  Add BOTH u and v to cover
  Remove all edges incident to u or v

Ratio: Always ≤ 2 × OPT
Why: We pick 2 nodes per edge, OPT needs at least 1
WEIGHTED VERTEX COVER (Pricing Method) Each edge has a "price" p(e).
A node is tight when Σp(e) = weight(v).
Raise prices until some node becomes tight.
Add all tight nodes to cover.

Fairness Lemma: For any vertex cover S, Σp(e) ≤ w(S)
Ratio: 2-approximation
GREEDY SET COVER Algorithm: While uncovered elements remain: pick the set S that covers the most uncovered elements per unit cost (w/|S∩R|).
Approximation Ratio: H(max set size) = 1 + 1/2 + 1/3 + ... (harmonic number)
🔍 Local Search — Hopfield Networks & State Flipping

Hopfield Neural Networks:

  • Graph G = (V_s, E_w) with vertex states ±1
  • Edge weights can be positive or negative
  • Good edge: w(u,v) × s(u) × s(v) < 0
  • Bad edge: w(u,v) × s(u) × s(v) > 0
  • Goal: maximize good edges (stable configuration)

State-Flipping Algorithm:

  • 1
    Start with any configuration
  • 2
    Find an "unsatisfied" node (bad edges outweigh good)
  • 3
    Flip that node's state
  • 4
    Repeat until stable (no unsatisfied nodes)
Terminates in ≤ Σ|w(e)| iterations
🎲 Randomization — Select & Hashing

Randomized Select (k-th largest):

Select(S, k): Pick splitter a_i ∈ S uniformly at random S⁻ = elements < a_i S⁺ = elements > a_i if |S⁻| = k-1: return a_i (found!) else if |S⁻| ≥ k: return Select(S⁻, k) else: // |S⁻| = t < k-1 return Select(S⁺, k-t-1)
Expected Time: O(n)

Universal Hashing:

  • Goal: Minimize collisions in hash table
  • Hash family H is universal if for x≠y: Pr[h(x)=h(y)] ≤ 1/m
  • Construction: h_a(x) = (Σaᵢxᵢ) mod p
  • where p is prime, aᵢ ∈ {0,...,p-1}
CHAINING Hash table with chaining: each slot is a linked list. Expected lookup: O(1+n/m) where n=keys, m=slots

Exam-Oriented Notes — Topper Style

MOST IMPORTANT
CO1: Network Flow
  • Max-Flow Min-Cut Theorem: Max flow = Min cut capacity
  • Ford-Fulkerson uses DFS; Edmonds-Karp uses BFS
  • Residual graph: forward edge c(u,v)-f(u,v), back edge f(u,v)
  • Augmenting path: path in residual graph from s to t
  • Max edges in residual graph = 2|E| (each edge can give forward + backward)
  • Source has no incoming edges; Sink has no outgoing edges
  • Flow conservation: at intermediate nodes, Σin = Σout
CO2: P, NP, NPC, PSPACE
  • P: Decidable in O(nᵏ) time
  • NP: Certificate verifiable in O(nᵏ) time
  • NPC: In NP + NP-Hard (every NP problem reduces to it)
  • NP-Hard: At least as hard as NPC; may not be in NP
  • PSPACE: Decidable using O(nᵏ) space (P ⊆ NP ⊆ PSPACE)
  • Q-SAT (Quantified Boolean Formula) is PSPACE-Complete
  • Polynomial reduction: A ≤ₚ B means "A is no harder than B"
  • If Y ≤ₚ X and Y is NP-Hard, then X is NP-Hard
CO3: Approximation Algorithms
  • Vertex Cover 2-approx: Pick both endpoints of any edge
  • Weighted VC pricing: Raise prices until node is tight, add tight nodes
  • Fairness Lemma: Σ_{e∈S} p(e) ≤ w(S) for any vertex cover S
  • Result: Algorithm gives ≤ 2×OPT (2-approximation)
  • Greedy Set Cover: Pick set maximizing coverage per unit cost
  • Set Cover ratio: H(d) where d = max set size, H = harmonic number
  • Load Balancing Greedy: Makespan ≤ 2×OPT*
CO4: Local Search
  • Hopfield network: nodes ∈ {-1, +1}, edges have weights
  • Good edge: w(u,v)·s(u)·s(v) < 0
  • Node is unsatisfied if Σ bad edges > good edges for it
  • State-flipping: flip unsatisfied node's state
  • Terminates: number of good edges strictly increases each flip
  • Bound: ≤ Σ|w(e)| iterations
  • Gradient descent local search: move to better neighbor; may get stuck
CO5: Randomization
  • Randomized Select: O(n) expected time for k-th order statistic
  • Good splitter: at least ¼ elements on each side → O(n) guaranteed
  • If always off-center: T(n) = T(3n/4) + n → O(n) still
  • Universal hashing: H is universal if Pr[h(x)=h(y)] ≤ 1/m for x≠y
  • Hash function: h_a(x) = (Σaᵢxᵢ) mod p, p prime
  • Chaining: Expected O(1+n/m) per lookup
  • Hash table with 9 slots: h(x) = x mod 9
CO6: Algorithm Design Approach
  • 5-step algorithm design checklist from exam Q1(b):
  • Step 1: Do you understand the problem clearly?
  • Step 2: Is there a simple algorithm/heuristic?
  • Step 3: Is it in the catalog (CSE3131/CSE4131)?
  • Step 4: Special cases you know how to solve?
  • Step 5: Which algorithm paradigm applies?
  • NPC Proof = (NP membership) + (NP-Hard via reduction)
  • Certificate = witness that verifies a YES answer in poly time
🎬 Movie Scheduling Problem — NPC Proof (High PYQ Chance!)

This appeared in the actual June 2023 end-sem exam (Q10)!

PROBLEM STATEMENT n movie projects. Each has start day and end day. You want maximum set of non-overlapping movies. Decide: Is there a set of size ≥ k?

Optimization version: Max non-overlapping intervals
DECISION PROBLEM Given I (intervals) and k: Is there a subset of ≥ k mutually non-overlapping intervals?

This is equivalent to Independent Set on Interval Graphs.
NPC PROOF SKETCH (via Independent Set reduction) 1. Membership in NP: Given a set S of intervals, verify non-overlap in O(k log k) time. ✓
2. NP-Hard: Reduce from Independent Set (known NPC).
   • For each vertex v, create an interval
   • For each edge (u,v), make intervals of u and v overlap
   • Independent set of size k ⟺ k non-overlapping intervals

EXAMPLE 03 — CO2: NPC Proof Show that Set Cover Decision Problem is in NPC.
1️⃣
Step 1: Show Set Cover ∈ NP
Certificate: A collection of sets S₁, S₂,...,Sₖ
Verifier: Check (1) |collection| ≤ k, (2) union = U. Both checkable in O(|U| × k) = polynomial time. ✓
2️⃣
Step 2: Show Set Cover is NP-Hard (Vertex Cover ≤ₚ Set Cover)
Given: Graph G=(V,E), integer k. Does G have a vertex cover of size ≤k?
Reduction: Universe U = E (all edges). For each vertex v, create set Sᵥ = {edges incident to v}. k stays same.
Then: G has vertex cover of size k ⟺ there exists k sets in {Sᵥ} covering all edges.
Reduction is poly time. ✓ So Set Cover is NP-Hard. ✓
Therefore: Set Cover is NP-Complete ✓
EXAMPLE 04 — CO3: Weighted Vertex Cover Apply pricing method on graph with vertices having costs 6, 5, 7, 5 (as in May 2024 PYQ).
→
Setup: Vertices a(6), b(5), c(7), d(5). Edges: (a,b)=0.7, (b,c)=5, (b,d)=∞, etc. Set all p(e)=0 initially.
→
Algorithm: While there exists an edge (i,j) where neither i nor j is tight:
- Select such edge (i,j)
- Increase p(i,j) until i or j becomes tight (Σ prices on incident edges = vertex cost)
- A node is tight when it has "used up" its budget
→
Result: S = set of all tight nodes = Vertex Cover
Quality guarantee: w(S) = Σ_{v∈S} weight(v) ≤ 2 × Σ p(e) ≤ 2 × OPT
This is a 2-approximation algorithm

EXAMPLE 07 — CO5: Universal Hash Verification Given U={a,b,c,d,e,f}, m=2, h₁(x), h₂(x) as in PYQ. Is H={h₁,h₂} a universal family?
→
H is universal if: For every x≠y in U, at most |H|/m = 2/2 = 1 hash function in H maps x and y to the same slot.
→
Check all pairs: For each pair (x,y), count how many of {h₁,h₂} give h(x)=h(y).
From the given table: check each pair manually.
If every pair has at most 1 collision among {h₁,h₂}: YES, universal
If any pair has 2 collisions: NO, not universal

QUESTION
What is the difference between tractable and intractable problems?
Click to reveal ▼
Tractable: Problems solvable in polynomial time O(nᵏ) — class P. Example: Sorting, MST, Max Flow.

Intractable: Problems believed to require super-polynomial (exponential) time in worst case. Class NP (when P≠NP). No known poly-time algorithm. Example: TSP, Vertex Cover, 3-SAT.

Key insight: Tractability is about worst-case polynomial-time solvability. NP problems can be verified polynomially but not necessarily solved.
QUESTION
What is a "certificate" in the context of NP problems?
Click to reveal ▼
A certificate (also called witness) is a proposed solution that can be verified in polynomial time.

Example for 3-SAT: Certificate = specific truth assignment to variables. We can check if it satisfies all clauses in O(n) time.

Significance of polynomial-time certifier: It defines the class NP. If a problem has a poly-time certifier, it's in NP. The certifier doesn't need to FIND the solution — just VERIFY it.
QUESTION
Why does Randomized Select run in O(n) expected time?
Click to reveal ▼
Key argument: A "good splitter" is one that puts at least n/4 elements on each side.

Probability of picking a good splitter = 1/2 (any element in middle 50% is good).

Expected number of trials to get a good splitter = 2 (geometric distribution).

With a good splitter, problem size reduces to ≤ 3n/4.

Recurrence: T(n) ≤ T(3n/4) + cn (with high probability)
Solving: T(n) = O(n) ✓
QUESTION
Can gradient descent local search always guarantee a global optimum?
Click to reveal ▼
NO! Gradient descent / local search cannot guarantee global optimum.

Why? It can get stuck in local optima — solutions that are better than all neighbors but not the best overall.

Analogy: Climbing the highest hill starting from a valley — you might reach a small hill first and get "stuck" thinking you're at the top.

Solutions: Simulated Annealing (accept worse solutions with some probability), Random Restarts (start from multiple initial solutions), Genetic Algorithms.
QUESTION
What is PSPACE and why is Q-SAT in PSPACE?
Click to reveal ▼
PSPACE: Class of decision problems solvable by a deterministic Turing Machine using polynomial SPACE (but possibly exponential time).

P ⊆ NP ⊆ PSPACE (SPACE is more powerful than TIME in terms of computability).

Q-SAT (Quantified Boolean Formula): Is ∃x₁∀x₂∃x₃... φ(x₁,x₂,...,xₙ) TRUE?

Q-SAT is in PSPACE because: we can evaluate the formula by trying all 2ⁿ assignments one at a time, reusing the same polynomial space. The key insight: we don't need to STORE all assignments simultaneously — just evaluate one at a time.
