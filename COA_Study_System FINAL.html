📘 Structural Components
1.1 Single Processor Computer — Main Components ▼

A single processor computer has four main structural components:

ComponentDefinition
CPU (Central Processing Unit)Controls operation and performs data processing. Contains ALU, Control Unit, and Registers.
Main MemoryStores data and programs currently being used. Volatile; loses data when power off.
I/O (Input/Output)Moves data between computer and external environment (keyboard, display, disk).
System InterconnectionMechanism (bus) that connects CPU, memory, and I/O and transfers data between them.
A single processor computer system consists of four main structural components: (1) Central Processing Unit (CPU), (2) Main Memory, (3) I/O, and (4) System Interconnection (Bus). The CPU controls all operations and performs data processing. Main memory stores programs and data in use. I/O provides communication with the external environment. The system interconnection links all components.

1.2 Multicore Computer System — Main Components ▼

A multicore system has multiple processors (cores) on a single chip. Each core has its own ALU, registers, and cache (L1/L2). They share L3 cache and main memory.

ComponentDefinition
Multiple CoresTwo or more processors (cores) on a single chip, each with its own ALU, CU, and registers.
L1 Cache (per core)Fastest, smallest cache private to each core. Holds frequently used data.
L2/L3 Cache (shared)Larger shared cache among cores; reduces main memory access.
Main MemoryShared RAM accessible by all cores.
I/O InterfaceHandles data exchange with peripherals.
Interconnect (NoC/Bus)Network connecting cores, cache, and memory.
A multicore computer system contains multiple processing cores on a single chip. Each core has its own registers and L1 cache. A shared L2/L3 cache is accessible by all cores. All cores share the main memory through a high-speed interconnect.

⚠️ Obstacles When Clock Speed & Logic Density Increase
Obstacles / Challenges ▼
ObstacleDescription
Power DissipationHigher clock = more switching = more heat. Requires expensive cooling.
RC DelayAs wires get thinner (high density), resistance and capacitance increase, slowing signals.
Memory LatencyCPU gets faster but memory doesn't keep up → memory wall problem.

As clock speed and logic density increase, four major obstacles arise: (1) Power dissipation increases leading to thermal issues; (2) RC delay increases as wire dimensions shrink; (3) Memory latency becomes a bottleneck as CPU outpaces memory speed; (4) Memory bandwidth limits data transfer rates.

📐 Key Formulas — MUST MEMORIZE
Little's Law: N = λ × T
N = avg items in system | λ = arrival rate | T = avg time in system
Amdahl's Law (Speedup): Sp = 1 / [(1-f) + f/N]
f = fraction enhanced | N = speedup factor of enhanced part
Effective CPI = Σ(IC_i × CPI_i) / Total IC
CPU Execution Time = IC × CPI × T_clock = IC × CPI / f_clock
MIPS = f_clock / (CPI × 10⁶) = IC / (T_exec × 10⁶)

🧠 Performance Balance & Moore's Law
Moore's Law + Performance Balance ▼

Moore's Law: The number of transistors on a chip doubles approximately every 18 months (or 2 years). Originally observed by Gordon Moore in 1965.

Moore's Law states that the number of transistors that can be placed on an integrated circuit doubles approximately every 18 months, leading to exponential growth in processor performance.

Performance Balance Problem: CPU speed grows ~60%/year, but DRAM access speed only grows ~7%/year. This creates a widening gap — the "Memory Wall."

Solutions to Performance Balance:

  • Increase DRAM bandwidth
  • Use cache memory hierarchy
  • Increase bus bandwidth
  • Use faster peripheral buses

🔁 Instruction Cycle
Instruction Cycle States (Without Interrupt) ▼
StateNameDescription
1Instruction Fetch (IF)Fetch instruction from memory using PC. Store in IR. Increment PC.
2Instruction Decode (ID)Decode the fetched instruction to determine operation and operands.
3Operand Fetch (OF)If needed, calculate operand address and fetch from memory/registers.

┌─────────────────────────────────────────────────┐ │ BASIC INSTRUCTION CYCLE │ └─────────────────────────────────────────────────┘ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │FETCH │───►│DECODE │───►│EXECUTE │───►│STORE │ │ │ │ │ │ │ │RESULT │ │PC→MAR │ │IR→CU │ │ALU op │ │ │ │MBR→IR │ │ │ │ │ │ │ │PC++ │ │ │ │ │ │ │ └────────┘ └────────┘ └────────┘ └────────┘ ▲ │ └─────────────────────────────────────────┘ (loop forever) Registers used: PC = Program Counter (holds address of next instruction) MAR = Memory Address Register MBR = Memory Buffer Register IR = Instruction Register CU = Control Unit
The instruction cycle without interrupts consists of: (1) Fetch — the instruction addressed by PC is loaded into IR and PC is incremented; (2) Decode — the control unit interprets the opcode in IR; (3) Execute — the indicated operation is performed; (4) Write-back — results are stored. This cycle repeats for each instruction.

In an instruction cycle with interrupts, after each instruction executes, the CPU checks for pending interrupts. If an interrupt occurs: (1) the current state (PC, registers) is saved; (2) the Interrupt Service Routine (ISR) address is loaded into PC; (3) the ISR executes; (4) the saved state is restored and normal execution resumes.
🔌 Microprocessor Bus Design Problems
Q4: 32-bit microprocessor, 32-bit instruction, first byte = opcode ▼
Assignment 2

32-bit instruction: 8-bit opcode (1st byte) + 24-bit operand/address

  • Max addressable memory = 2^24 bytes = 16 MB
  • 32-bit address bus + 16-bit data bus: Address space full (2^32), but each bus cycle fetches only 16 bits → need 2 cycles for 32-bit instruction → SLOWER
  • 16-bit address bus + 16-bit data bus: Address space reduced to 2^16 = 64KB → severely limits addressable memory → MUCH SLOWER & LIMITED
  • PC needs 24 bits (to hold operand address). IR needs 32 bits (full instruction).
a. Max addressable memory = 2^24 = 16 MB (24-bit address field). b. A 16-bit data bus requires 2 bus cycles to fetch a 32-bit instruction, reducing throughput. A 16-bit address bus limits addressable memory to 64KB, severely degrading performance. c. PC requires 24 bits; IR requires 32 bits.
Q5: 32-bit vs 16-bit microprocessor — Calculate Improvement ▼
Assignment 2

Mix: 20% are 32-bit, 40% are 16-bit, 40% are 8-bit (same bus cycle duration)

  • 16-bit μP bus cycles per operand: 32-bit needs 2 cycles, 16-bit needs 1, 8-bit needs 1. Avg = 0.2×2 + 0.4×1 + 0.4×1 = 0.4+0.4+0.4 = 1.2 cycles
  • 32-bit μP bus cycles per operand: 32-bit needs 1 cycle, 16-bit needs 1, 8-bit needs 1 (padded). Avg = 1.0 cycle
  • Improvement = 1.2/1.0 = 1.2× improvement (20% faster)
The 16-bit processor requires on average 1.2 bus cycles per fetch (since 32-bit items need 2 cycles). The 32-bit processor requires only 1 cycle per fetch on average. The improvement ratio = 1.2/1.0 = 1.2 (i.e., 20% improvement).

Cache Memory — CO4 · Direct / Associative / Set-Associative

📐 Cache Address Formula Sheet
Address = TAG + INDEX(LINE) + WORD(OFFSET)
OFFSET bits = log₂(Block Size in bytes) | LINE bits = log₂(Number of Lines/Sets) | TAG = remaining bits
Direct Mapped: #Lines = Cache Size / Block Size
Tag bits = Address bits − Line bits − Offset bits
Set Associative: #Sets = #Lines / Associativity
Index bits = log₂(#Sets) | Tag = Addr − Index − Offset
Fully Associative: Index = 0 bits | Tag = Address bits − Offset bits
⚙️ Memory Hierarchy
Why Use Memory Hierarchy? + Principle of Locality ▼

Reason for Memory Hierarchy: No single memory technology can be simultaneously fast, large, and cheap. We use a hierarchy where each level is faster (but smaller and costlier) than the level below it.

┌─────────┐ Fastest, Smallest, Costliest │Registers│ ~1 cycle └────┬────┘ ┌────▼────┐ │ L1 $ │ ~4 cycles └────┬────┘ ┌────▼────┐ │ L2 $ │ ~12 cycles └────┬────┘ ┌────▼────┐ │ L3 $ │ ~40 cycles └────┬────┘ ┌────▼────┐ │ RAM │ ~100+ cycles └────┬────┘ ┌────▼────┐ │ Disk │ ~millions of cycles └─────────┘ Slowest, Largest, Cheapest

Principle of Locality:

  • Temporal Locality: Recently accessed data is likely to be accessed again soon (loops).
  • Spatial Locality: Data near recently accessed data is likely to be accessed (arrays).
Memory hierarchy exists because no single technology offers high speed, large capacity, and low cost simultaneously. By organizing memory in levels (registers → L1 cache → L2 → RAM → disk), we exploit the principle of locality — temporal (recently used data reused) and spatial (nearby data also accessed) — to achieve near-register speed at near-disk cost.
🧮 GATE-Level Cache Problems — Fully Solved
Q9 (GATE 2017): Direct-mapped TAG=10bits → 16-way set-assoc TAG=? ▼
GATE 2017Assignment 2
  • Direct-mapped: TAG=10, so Line bits = Addr − 10 − Offset. Let addr=A, offset=O. Line bits = A−10−O
  • 16-way set-associative: #sets = N/(B×16). Index bits = Line bits − log₂(16) = (A−10−O) − 4
  • New TAG = A − new_index − O = A − (A−10−O−4) − O = 14 bits
Answer: 14 bits. When going from direct-mapped to 16-way set-associative cache (with same capacity N and block size B), the number of sets reduces by factor 16. So index bits decrease by log₂(16) = 4 bits, and TAG field increases by 4 bits: 10 + 4 = 14 bits.

Hamming Code — Error Detection & Correction ▼
PYQ EndSem

Hamming code adds check bits at positions that are powers of 2 (1,2,4,8...) to detect and correct single-bit errors.

For data bits d, check bits r must satisfy: 2^r ≥ d + r + 1

PYQ: 12-bit Hamming word, 8 data + 4 check bits, find original data

Position map: C1 C2 D1 C4 D2 D3 D4 C8 D5 D6 D7 D8 (C=check, D=data)

Data bits: d8=0, d7=1, d6=0, d5=1, d4=1, d3=1, d2=0, d1=1

Check bits (old): C8=0, C4=0, C2=0, C1=0. Compute syndrome to find error position and correct it.

Hamming Code Quick Steps:

  1. Parity bits at positions that are powers of 2: P1, P2, P4, P8...
  2. Fill remaining positions with data bits in order
  3. P1 covers bits 1,3,5,7,9,11... (positions with bit 1 set in binary)
  4. XOR all parity checks → result gives error position (syndrome)

The three key concepts of von Neumann architecture are: (1) Data and instructions are stored in a single read-write memory; (2) Contents of memory are addressable by location, without regard to the type of data; (3) Execution occurs in a sequential manner (instruction at next sequential address is fetched and executed) unless explicitly modified by a jump or branch instruction.

MEDIUMPYQ EndSem Q7a: Performance Balance — explain and give 2 solutions ▼
Performance balance refers to the mismatch between CPU speed and memory/I/O speed. CPU performance has grown at ~60% per year while DRAM access speed improves only ~7% per year, creating a "memory wall." Solutions: (1) Increase DRAM bandwidth — use wider buses, multiple data buses, or interleaved memory; (2) Use cache memory — place high-speed SRAM between CPU and DRAM to exploit locality of reference and reduce average memory access time.
MEDIUMPYQ Q3a: Handle multiple interrupts — flow chart approach ▼

Two approaches to handle multiple interrupts:

1. Sequential (Disable Interrupts): While handling one interrupt, disable all others. After ISR completes, re-enable and check for pending interrupts. Simple but lower-priority interrupts must wait.

2. Nested Interrupts (Priority-Based): Higher-priority interrupts can interrupt a lower-priority ISR. Uses a priority mask. Allows real-time response to critical events.

Sequential: Priority-Based: IRQ1→ISR1 IRQ1(low)→ISR1 (IRQ2 waits) IRQ2(high)→interrupts ISR1 ISR1 done→ISR2 →ISR2 runs →returns to ISR1 →ISR1 completes

Q1: Main structural components of single processor computer system ▼
A single processor computer system has four main structural components: (1) CPU — Contains the ALU for computations, Control Unit for directing operations, and Registers for temporary storage. (2) Main Memory — Stores data and programs currently being executed; volatile storage. (3) I/O — Provides interface between computer and external devices (keyboard, monitor, disk). (4) System Interconnection — The bus structure that connects CPU, memory, and I/O, allowing data transfer between components.
Q2: Multicore computer system components ▼
A multicore computer system contains: (1) Multiple Processor Cores — each with its own ALU, control unit, and registers, enabling parallel execution. (2) L1 Cache — Per-core private cache (typically 32–64 KB) for fastest data access. (3) L2/L3 Shared Cache — Larger cache shared among cores to reduce main memory accesses. (4) Main Memory (RAM) — Shared DRAM accessible by all cores. (5) I/O Interface — Connects to peripherals via PCIe, USB, SATA buses. (6) Interconnect (Bus/NoC) — High-speed connection linking cores, cache, and memory.
Q3: Differentiate (i) Microprocessor vs Microcontroller (ii) Organization vs Architecture (iii) Embedded vs Deeply Embedded ▼

(i) Microprocessor vs Microcontroller

A microprocessor is a CPU on a single chip requiring external memory and I/O chips (e.g., Intel 8086). A microcontroller integrates CPU, RAM, ROM, and I/O ports on one chip, designed for embedded control applications (e.g., ATmega328 in Arduino). Microcontrollers are cheaper, consume less power, and are used in appliances, IoT devices, and robotics.

(ii) Computer Organization vs Architecture

Computer Architecture refers to the logical design visible to the programmer — instruction set, data types, addressing modes, registers, and interrupts. Computer Organization refers to the physical implementation of the architecture — control signals, memory technology, bus design, and ALU circuit details. Two machines with the same architecture can have different organizations.

(iii) Embedded vs Deeply Embedded

An embedded system is a computer system designed for a specific function within a larger device, possibly with a simple user interface (e.g., ATM, printer). A deeply embedded system has extreme constraints on power, size, and cost, operates autonomously with no user interface, and often runs bare-metal (no OS). Examples: cardiac pacemakers, airbag controllers, RFID chips.
Q4: Cloud Computing Services with Diagram ▼
Cloud computing delivers computing services over the internet on-demand. Three service models: (1) Infrastructure as a Service (IaaS) — provides virtualized hardware (VMs, storage, networking). Users manage OS and above. Example: AWS EC2. (2) Platform as a Service (PaaS) — provides development platform including OS, runtime, middleware, and databases. Developers focus on application code. Example: Google App Engine. (3) Software as a Service (SaaS) — delivers complete applications over the internet. Users just use the software. Example: Gmail, Salesforce.
Q5: Techniques to increase microprocessor speed ▼
Five techniques to increase microprocessor speed: (1) Increase Hardware Speed — use faster switching technology (GaAs) and higher clock frequencies. (2) Increase Cache Size — more on-chip cache reduces expensive main memory accesses. (3) Change Processor Architecture — pipelining, superscalar, and out-of-order execution allow multiple instruction executions simultaneously. (4) Use Multiple Processors — multicore and multiprocessor systems enable parallel execution of programs. (5) Reduce Memory Access Time — wider buses, memory interleaving, and DMA reduce the memory bottleneck.
Q6: Obstacles when clock speed and logic density increase ▼
As clock speed and logic density increase, four key obstacles arise: (1) Power Dissipation — higher switching frequency increases heat generation, requiring costly cooling solutions. (2) RC Delay — as wires become thinner with higher density, resistance and capacitance increase, slowing signal propagation. (3) Memory Latency — CPU speed grows faster than memory access speed, creating the "memory wall" bottleneck. (4) Memory Bandwidth — even with acceptable latency, the rate of data transfer becomes the limiting factor in system performance.

Q16: VAX 11/780 (5MHz, 1MIPS) vs IBM RS/6000 (25MHz, 18MIPS). Instruction count ratio & CPI? ▼

Let IBM CPU time = x, VAX CPU time = 12x

MIPS = IC / (CPU_time × 10⁶)

VAX: 1 = IC_VAX / (12x × 10⁶) → IC_VAX = 12x × 10⁶

IBM: 18 = IC_IBM / (x × 10⁶) → IC_IBM = 18x × 10⁶

IC_VAX / IC_IBM = 12x / 18x = 2/3

VAX CPI = (5×10⁶ × 12x) / (12×10⁶ × x) = 5

IBM CPI = (25×10⁶ × x) / (18×10⁶ × x) = 25/18 ≈ 1.39

Q1: Von Neumann 3 Key Concepts ▼
The three key concepts of von Neumann architecture: (1) Stored Program — data and instructions are stored in a single read-write memory, allowing programs to be treated as data and modified. (2) Single Memory Space — both data and instructions share the same addressable memory space. (3) Sequential Execution — instructions are fetched and executed one at a time in sequence, unless a branch instruction alters the flow.
Q2: Instruction Execution States (without interrupt) ▼
The states defining instruction execution without interrupt are: (1) Instruction Fetch (IF) — fetch instruction from address in PC into IR; increment PC. (2) Instruction Decode (ID) — CU decodes opcode in IR to determine operation and operand locations. (3) Operand Fetch (OF) — if required, calculate effective address and fetch operand from memory or register. (4) Execute (EX) — perform the operation (arithmetic, logic, data transfer, control). (5) Result Store (WB) — write result back to register or memory location.
Q6: Reasons for using a memory hierarchy ▼
A memory hierarchy is used for the following reasons: (1) Cost-Performance Trade-off: Fast memory (SRAM) is expensive. Slow memory (DRAM/Disk) is cheap. Hierarchy gives fast speed at low average cost. (2) Principle of Locality: Programs exhibit temporal and spatial locality. A small fast memory can hold frequently used data. (3) Reduced Average Access Time: Most accesses are served by cache (fast), rarely going to main memory or disk. (4) Scalability: Different levels can be optimized independently.

📌 Key Definitions (1-2 lines each)

Microprocessor

CPU on single chip; needs external RAM/ROM. Examples: Intel Core, AMD Ryzen.

Microcontroller

CPU + RAM + ROM + I/O on single chip. For embedded apps. Example: 8051, ATmega.

Computer Architecture

Attributes visible to programmer — ISA, data types, addressing modes.

Computer Organization

Physical implementation of architecture — control signals, bus width, memory tech.

Von Neumann

Stored program + single memory space + sequential execution.

Moore's Law

Transistor count doubles every 18 months.

Little's Law

N = λT. Items in system = arrival rate × avg time in system.

Amdahl's Law

Overall speedup limited by fraction NOT enhanced. Sp = 1/[(1-f)+f/N].

Temporal Locality

Recently accessed data will likely be accessed again soon.

Spatial Locality

Data near recently accessed data is likely to be accessed.

Direct Mapped Cache

Each memory block maps to exactly one cache line. Fast but high conflict misses.

Set Associative Cache

Memory block maps to a set; any line within that set can hold it. Balance of direct and fully associative.

Fully Associative Cache

Any block can go to any cache line. No index field. Best hit rate, slowest to search.

IEEE 754 Single Precision

32 bits: S(1) E(8) F(23). Value = (−1)^S × 2^(E−127) × 1.F. Bias = 127.

Booth's Algorithm

Signed binary multiplication. Bit pair 00/11 → no op; 01 → add; 10 → subtract. Arithmetic right shift each step.

Half Adder

S = A ⊕ B, C = A·B. Adds 2 bits.

Full Adder

Sum = A⊕B⊕Cin, Cout = AB+BCin+ACin. Adds 3 bits including carry-in.
